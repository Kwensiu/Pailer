# Settings - Personalization

The Settings page is your control center for customizing Pailer's behavior and appearance. Through rich configuration options, you can adjust various aspects of the application according to your personal needs and preferences.

## Page Overview

The Settings page is divided into multiple configuration categories:

- **Automation Settings**: Configure auto-update, cleanup and other functions
- **Management Configuration**: Scoop core configuration and path settings
- **Security Settings**: Virus scanning and security-related configurations
- **Interface Settings**: Theme, language, startup and other interface customizations
- **System Settings**: System startup, hotkeys and other system integrations

## Basic Usage

### 1. Navigate Settings Categories

The page uses tabs to organize different setting categories:

- **Tab Switching**: Click top tabs to switch setting categories
- **Quick Location**: Settings are saved automatically, no manual saving needed
- **Reset Function**: Some settings support reset to default values

### 2. Modify Settings

Most settings can be modified directly:

- **Instant Effect**: Most settings take effect immediately after modification
- **Restart Required**: Some settings need application restart to take effect
- **Confirmation Prompts**: Important setting modifications have confirmation prompts

### 3. Settings Search

Use search function to quickly find needed settings:

- **Keyword Search**: Enter setting names or related keywords
- **Category Filtering**: Quick filter by setting categories
- **Recent Usage**: Display recently modified settings

## Detailed Setting Categories

### Automation Settings

Configure various automation functions:

- **Auto Update**: Set frequency of automatic software check and update
- **Auto Cleanup**: Configure automatic cleanup of cache and temporary files
- **Scheduled Tasks**: Set regular system maintenance tasks
- **Notification Settings**: Configure notification methods for automation operations

### Management Configuration

Scoop's core management configuration:

- **Scoop Path**: Set Scoop installation path
- **Bucket Configuration**: Configure default buckets and bucket management strategies
- **Proxy Settings**: Set network proxy configuration
- **Update Strategy**: Configure software update policies and rules

### Security Settings

Configurations to enhance system security:

- **VirusTotal Integration**: Configure VirusTotal API for virus scanning
- **Download Verification**: Set integrity verification for software packages
- **Permission Control**: Configure operation permissions and security policies
- **Audit Logging**: Enable operation log recording

### Interface Settings

Customize application appearance and behavior:

- **Theme Selection**: Choose light, dark or system theme
- **Language Settings**: Switch interface language (Chinese/English)
- **Startup Page**: Set default page displayed when application starts
- **Interface Layout**: Adjust layout and display of interface elements

### System Settings

Settings integrated with operating system:

- **Startup Launch**: Configure whether to start automatically with system
- **System Tray**: Set system tray icon and behavior
- **Hotkey Settings**: Configure global shortcuts
- **File Association**: Set file type associations

## Advanced Features

- **Export Configuration**: Export current settings as file

- **Sync Settings**: Sync settings between different devices
- **Version Management**: Manage historical versions of configuration

### Advanced Options

Configurations for advanced users:

- **Debug Mode**: Enable debug functions for detailed information

- **Experimental Features**: Enable experimental new features

## Common Issues

### Settings not taking effect after modification?

1. **Restart Application**: Some settings need application restart to take effect
2. **Check Dependencies**: Ensure system meets setting dependencies
3. **Clear Cache**: Clear application cache to reset setting state
4. **Reset Settings**: Try resetting to default values and reconfigure

### How to backup settings?

1. **Export Function**: Use export function in settings page
2. **Manual Backup**: Backup configuration file directory
3. **Cloud Sync**: Use cloud services to sync settings files
4. **Regular Backup**: Establish regular backup habits

### Restore default settings?

1. **Reset Button**: Use reset function in settings page
2. **Selective Reset**: Only reset specific category of settings
3. **Complete Reset**: Reset all settings to default values
4. **Backup Recovery**: Recover settings from backup files

## Best Practices

### Settings Management Strategy

- **Progressive Configuration**: Gradually configure and test various settings
- **Record Changes**: Record important setting changes
- **Regular Review**: Regularly review and optimize setting configuration

### Security Recommendations

- **Configure Cautiously**: Be cautious when modifying security-related settings
- **Regular Updates**: Keep security settings up-to-date
- **Backup Important Settings**: Backup important security configurations

## Related Features

The Settings page is closely related to other functions:

- **All Pages**: Settings affect behavior of all pages
- **Doctor Page**: Check setting-related system issues
- **Installed Page**: Settings affect software management behavior
- **Bucket Page**: Settings affect bucket management strategies

Through the Settings page, you can completely customize Pailer's behavior, creating a software package management environment that meets your personal needs.
