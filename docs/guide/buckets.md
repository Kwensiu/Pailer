# 仓库 - 管理源

仓库页面是您管理 Scoop 软件源的控制中心。通过这个页面，您可以添加、更新和管理所有软件仓库，确保能够访问到最新的软件。

## 页面概览

仓库页面提供了完整的仓库管理功能：

- **仓库列表**：显示所有已安装的仓库及其状态
- **仓库搜索**：搜索本地仓库和在线社区仓库
- **批量操作**：同时更新多个仓库，显示详细进度
- **分支管理**：管理每个仓库的Git分支
- **详细信息**：查看仓库的详细信息和统计数据

## 功能特性

### 仓库显示

- **仓库名称**：仓库的唯一标识符
- **更新状态**：显示仓库是否需要更新
- **应用数量**：仓库中包含的软件数量
- **最后更新**：仓库的最后更新时间
- **Git 信息**：仓库的 Git 分支和远程地址
- **分支选择器**：对于Git仓库，可以切换分支

### 更新操作

- **单个更新**：点击仓库卡片上的更新按钮
- **批量更新**：使用顶部的"更新所有Git仓库"按钮
- **进度显示**：实时显示每个仓库的更新进度和结果
- **更新结果**：显示每个仓库的更新成功/失败状态

### 搜索功能

- **本地仓库搜索**：搜索已安装的仓库
- **社区仓库搜索**：搜索GitHub上的社区仓库
- **排序选项**：按stars、forks、应用数量等排序
- **搜索过滤**：设置最小stars、禁用中文仓库等过滤条件

## 高级使用

### 社区仓库搜索

- **启用社区搜索**：点击"社区仓库"按钮启用在线搜索
- **搜索配置**：设置最小stars数量、禁用中文仓库等
- **缓存管理**：社区仓库搜索使用缓存，提高搜索速度
- **安装仓库**：直接从搜索结果安装新仓库

### 分支管理

- **查看分支**：点击分支选择器查看所有可用分支
- **切换分支**：切换到不同的分支获取不同版本的软件
- **分支更新**：切换分支后自动更新仓库内容
- **分支状态**：显示当前活跃分支

### 批量更新管理

- **更新队列**：显示所有待更新的仓库
- **进度跟踪**：实时显示每个仓库的更新进度
- **结果汇总**：显示更新成功和失败的统计
- **错误处理**：显示更新失败的详细错误信息

### 仓库详情

- **仓库信息**：仓库名称、描述、URL等基本信息
- **统计数据**：stars数量、forks数量、应用数量
- **Git信息**：远程地址、当前分支、最后更新时间
- **应用列表**：仓库中的所有软件包列表

## 相关功能

仓库页面与其他功能紧密相关：

- **搜索页面**：搜索结果依赖于已安装的仓库
- **已安装页面**：显示来自各个仓库的已安装软件
- **设置页面**：配置仓库相关的设置选项

通过仓库页面，您可以有效管理软件源，确保能够访问到丰富且最新的软件资源。
